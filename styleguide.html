Warranty Reviewer Design System

A pared-down system for the agentic warranty-claims demo. Toyota brand red is reserved for the single thing on screen the reviewer should look at next — flagged claims, primary CTAs, AI-insight accents. Everything else lives on a warm-neutral gray ladder.

One filled red primary per surface — the destructive Reject CTA. Approve and Escalate stay outlined.

Filled ink = neutral primary (Save). Outlined = secondary.

Cards

White surface, 1px toyota-200 border, 10px radius. No drop shadows — borders carry the elevation.

Standard panel

Used for Metadata, Policy Flags, Parts Detail. Section heading uses the uppercase-tracked label pattern shown above.

Empty / muted state

Dashed border + tinted surface signals "nothing here yet" without using a status color.

Explainer cards

Used on onboarding / empty-state screens to explain what each agent does. Three-up layout, equal height, brand-red icon glyph.

Policy Validation

Checks every claim against ~30 warranty rules. Flags hours over threshold, out-of-warranty mileage, unauthorized parts. Returns pass/fail per rule with severity.

AI Summary

Reads the technician notes plus validation output and produces a three-sentence reviewer narrative with a recommended action and confidence score.

Anomaly Detection

Spots statistical outliers across dealers — repeat claim codes, unusual cost spikes, suspicious labor patterns. Surfaces patterns a single-claim review would miss.
